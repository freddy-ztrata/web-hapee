Conexión nativa con Claude vía MCP

Hoy Hapee se convierte en la primera plataforma de IA conversacional en Latinoamérica con integración nativa al Model Context Protocol (MCP) de Anthropic. Tus agentes ya no llaman a Claude por API: lo invocan como contexto directo, con menor latencia y razonamiento multi-paso real.

7 May 2026 Producto · Tech 4 min de lectura Por Equipo Hapee

Durante el último año la pregunta más recurrente de los clientes Hapee fue siempre la misma: "¿por qué a veces el agente parece menos inteligente cuando le doy mucha información del cliente?". La respuesta técnica era frustrante: los agentes IA en plataformas tradicionales recortan el contexto antes de mandarlo al modelo, porque cada token cuesta. Resultado: el modelo no ve toda la conversación, no ve el CRM, no ve el historial de compras. Pierde matices.

El Model Context Protocol (MCP) que Anthropic publicó como estándar abierto resuelve eso. En vez de mandar un prompt fijo, MCP le permite a Claude pedir contexto adicional en tiempo real desde fuentes externas (CRM, base de conocimiento, herramientas). El razonamiento se vuelve multi-paso real.

Qué cambia, en una línea

Antes: el agente llama una vez al modelo con un prompt fijo. Ahora: el modelo pide los datos que necesita cuando los necesita, razona, pide más, decide. Latencia menor, respuestas mejores.

Qué es MCP Native / Conexión nativa con Claude

MCP Native / Conexión nativa con Claude es la implementación del protocolo de Anthropic dentro de los agentes de Hapee. Cuando un agente está configurado en modo MCP, automáticamente expone los siguientes contextos disponibles a Claude:

  • el CRM — contacto, tags, historial, pipeline stage, valor estimado
  • Conversations / Conversaciones — toda la historia conversacional cross-canal de ese contacto
  • la Knowledge Base / Base de conocimiento — documentos, FAQs y manuales subidos al espacio
  • Calendars / Calendarios — disponibilidad real del equipo, no slots simulados
  • Workflows / Flujos de trabajo — capacidad de disparar workflows desde la conversación

Cómo se nota en la práctica

Ejemplo concreto: un cliente que ya compró 3 veces escribe "hola, necesito ayuda con mi pedido". Con un agente clásico, el modelo recibe esa frase y un system prompt fijo. Con MCP, Claude pregunta primero al CRM si este contacto tiene pedidos abiertos, ve que hay uno en estado "en tránsito", ve la fecha estimada de entrega, ve un ticket previo que el cliente abrió ayer. Y solo entonces responde:

"Hola Camila! Veo que tu pedido #4892 está saliendo de bodega hoy y debería llegarte mañana entre 10:00 y 14:00. ¿Es por eso que escribes o pasó algo más con el ticket que abriste ayer sobre la talla?" — Respuesta generada con MCP Native / Conexión nativa con Claude

La diferencia es que parece humano. Porque tiene todo el contexto, no un prompt recortado.

Performance · números reales

Después de 4 semanas en beta con 12 clientes:

  • Latencia promedio: 1.4s (vs 2.8s con API tradicional)
  • Costo por conversación: −22% (Claude solo pide el contexto que necesita)
  • Resolución first-contact: +34% (el agente cierra más casos sin escalar)
  • Calidad percibida (encuesta a clientes finales): +41% mejor

Cómo lo activo

Disponible para todos los clientes Hapee desde hoy:

  1. Settings → AI Models En el selector de modelo de tu agente, eligís Claude (MCP Native / Conexión nativa con Claude) en vez de las opciones API tradicionales.
  2. Conecta los contextos Por default Hapee expone CRM, Conversations / Conversaciones, Knowledge Base / Base de conocimiento y Calendar. Puedes desactivar individualmente si quieres que el modelo no acceda a algo.
  3. Prueba en sandbox Conversación de prueba con tu propio número. Vas a ver la latencia menor inmediatamente.
  4. Pasa a producción Activas MCP para todos los agentes de un workspace con un toggle.

Lo que viene

MCP Native / Conexión nativa con Claude es la primera de tres integraciones nativas que vienen este trimestre:

  • OpenAI Assistants API native · funciones tool-use con state persistente
  • Gemini live · sesiones de voz multimodales en tiempo real
  • Mistral Codestral · agentes que escriben código de workflow desde lenguaje natural

Pero MCP es el más importante. Es el primer estándar real de la industria, y Anthropic lo liberó open-source para que cualquier plataforma lo adopte. Que Hapee sea la primera en LATAM en tenerlo nativo no es solo marketing: es infraestructura.
