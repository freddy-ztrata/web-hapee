Fable Mythos llega a Hapee: más poder a la conexión MCP

Anthropic acaba de presentar Fable Mythos, una nueva incorporación a la familia de modelos Fable de Claude. En Hapee ya estamos preparando su integración para sumarla a la conexión nativa que tenemos con Claude vía Model Context Protocol — el mismo stack con el que hoy nuestros agentes razonan, recuperan contexto y resuelven conversaciones en producción.

10 Jun 2026 Producto · Tech 3 min de lectura Por Equipo Hapee

Hace un mes liberamos la conexión nativa con Claude vía MCP: los agentes Hapee dejaron de llamar a Claude por API y pasaron a invocarlo como contexto directo. Menor latencia, razonamiento multi-paso real, mejor uso del contexto del cliente. Esa base es la que hoy hace posible que sigamos sumando capacidades sin reescribir nada.

Esta semana Anthropic introdujo Fable Mythos, una nueva variante de su familia Fable. Es muy reciente — lleva apenas días disponible — y todavía estamos en proceso de probarla a fondo. Pero la dirección está clara: se va a sumar al stack de modelos Fable que ya usamos en Hapee para potenciar el razonamiento de los agentes.

Qué es la familia Fable y cómo la usamos

Hapee ya integra varios modelos Fable como parte de su capa de razonamiento. Cada cliente puede elegir el que mejor le acomoda a su caso de uso, y los agentes pueden alternar entre ellos según la tarea: clasificación rápida, conversación larga, razonamiento estructurado, recuperación de contexto desde el CRM, lectura de la base de conocimiento, etcétera.

La ventaja de tenerlos todos detrás del mismo servidor MCP de Hapee es que el cambio entre modelos es invisible para el cliente final y, para el equipo que opera el agente, es un toggle en la configuración. No hay que reescribir prompts ni mover datos.

Qué suma Mythos al stack

Mythos se está perfilando como una variante orientada a continuidad y razonamiento prolongado en conversaciones largas. En la práctica, eso significa agentes que mantienen mejor el hilo cuando el cliente cambia de tema, vuelve a una conversación anterior, o entra en un embudo de varios turnos.

No prometemos benchmarks específicos: el modelo es muy nuevo. Lo que sí podemos decir es que la arquitectura MCP de Hapee está diseñada para que, apenas Mythos esté listo para producción, los agentes existentes lo puedan adoptar sin migraciones complejas. Es la misma conexión, con un modelo más.

Lo importante

Hapee no depende de un solo modelo. La conexión nativa vía MCP nos permite incorporar cada nueva variante de Claude — incluido Fable Mythos — apenas Anthropic la libera, sin tocar la lógica de los agentes que ya operan en producción.

El rol de Claude Code en el camino

Internamente, el equipo de Hapee usa Claude Code para iterar sobre el servidor MCP, mantener integraciones, escribir documentación y probar nuevas variantes apenas salen. Es parte de lo que nos permite estar listos para incorporar modelos nuevos en cuestión de días, no semanas.

Esto no es teoría: es el patrón con el que construimos la conexión MCP original, el que estamos usando ahora con Mythos, y el que vamos a seguir usando con cada variante que venga.

Qué cambia para los clientes Hapee hoy

En lo inmediato: nada que tengas que hacer. Si ya operas tu agente Hapee con la conexión MCP nativa, estás en la misma infraestructura que recibirá Mythos cuando esté listo para uso general. Vamos a comunicarlo en este mismo blog y en el cambio de versión del producto.

Para los clientes que todavía no migraron a la conexión MCP, el camino sigue siendo el mismo: activar MCP en su workspace, conectar contexto (CRM, conversaciones, base de conocimiento, calendario) y dejar que los agentes razonen con la información completa. Cada modelo Fable que sumamos al stack opera sobre esa misma base.
